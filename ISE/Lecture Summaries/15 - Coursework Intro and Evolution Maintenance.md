# 15 - Coursework Intro and Evolution Maintenance

Source: [15 - Coursework Intro and Evolution Maintenance.pdf](<../Lecture Slides/15 - Coursework Intro and Evolution Maintenance.pdf>)

## Core Summary

This lecture links coursework-style evidence to professional engineering practice and introduces evolution/maintenance themes. It emphasises traceability, issues, branches, merge requests, code reviews, conventions, comments, tests, and documentation.

## Good Engineering Evidence

Good evidence connects:
- issues/tickets;
- requirements;
- markdown documentation;
- code comments;
- tests;
- branches;
- merge requests;
- code reviews.

Traceability matters because the team should be able to explain why work exists, how it was implemented, and how it was tested.

## Maintainability Practices

- coding conventions;
- naming conventions;
- code reviews;
- issue tracking;
- branching;
- merge request templates;
- tests;
- documentation.

## Evolution Link

Maintenance/evolution requires understanding existing behaviour and changing it safely. Weak traceability makes later changes more expensive.

## Exam Angles

- Use traceability as a key word.
- Explain how issues, tests, branches, and reviews provide evidence of good practice.
- Connect maintainability to evolution and maintenance cost.
