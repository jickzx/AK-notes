# 17 - Software Quality

Source: [17 - Software Quality.pdf](<../Lecture Slides/17 - Software Quality.pdf>)

## Core Summary

This lecture argues that software quality is pervasive. Quality cannot be left to final testing; it must be planned, managed, reviewed, and improved throughout the process.

## Quality Is Pervasive

Quality activities include:
- requirements validation;
- specification inspections;
- prototype review;
- coding standards;
- code reviews;
- pair programming;
- test planning;
- release testing;
- documentation standards;
- traceability.

## QA Team Responsibilities

QA teams may:
- plan for quality;
- define standards and procedures;
- check projects conform to company standards;
- identify inspection points;
- identify useful quality measures;
- improve testing protocols;
- document lessons learned.

QA teams should ideally be separate from development teams and report above the project manager so quality is not sacrificed to short-term schedule pressure.

## Quality Plan

A quality plan can include:
- product introduction;
- product plans;
- process description;
- quality goals;
- risks and risk management.
