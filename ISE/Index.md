## Best Revision Route

1. Read [COMP1003 Exam Guide.md](<COMP1003 Exam Guide.md>).
2. Skim [Glossary.md](<Glossary.md>) for terms you cannot define cleanly.
3. Read [Required Reading Notes/02 - Reweighted Exam Priorities After Required Reading.md](<02 - Reweighted Exam Priorities After Required Reading.md>).
4. Work through [Revision Notes](<Revision Notes>) in order.
5. Use [Exam and Revision/Practice Questions](<Exam and Revision/Practice Questions>) to test recall.
6. Use [Exam and Revision/Past Papers](<Exam and Revision/Past Papers>) for timing and answer shape.
7. Return to cited lecture slides only where a note feels weak or unclear.

## Practice Paper Drill Method

For each practice or mock question:

1. Answer from memory first, timed at about 2.4 minutes per mark.
2. Check the answer against [COMP1003 Exam Guide.md](<COMP1003 Exam Guide.md>) and [Revision Notes/07 - Exam Answer Patterns.md](<Revision Notes/07 - Exam Answer Patterns.md>).
3. If you missed a definition, add it to [Glossary.md](<Glossary.md>).
4. If you missed an answer structure, add it to [Revision Notes/07 - Exam Answer Patterns.md](<Revision Notes/07 - Exam Answer Patterns.md>).
5. If you missed a planning calculation, redo it until you can identify dependencies, critical path, slack, and staff conflicts without looking.

## Highest Priority Topics

- Requirements engineering: failure causes, elicitation, stakeholders, personas, use cases, validation, non-functional requirements.
- Testing: unit, integration, release, acceptance, user testing, TDD, CI/CD.
- Maintainability: coding conventions, code reviews, pair programming, refactoring, technical debt.
- Specifications, UML, and prototyping: clear specs, activity diagrams, sequence diagrams, prototypes.
- Agile: Agile Manifesto values, Scrum, XP, Kanban, agile vs traditional, hybrid reasoning.
- Quality, risk, and planning: QA beyond testing, software/project risks, risk strategies, Gantt/PERT/critical path/staffing.
- Scenario enrichers: usability, accessibility, authorization/security, user involvement.

## Quick Exam Rule

For most short-answer questions, write one clear point per mark. If a question asks for bullet points, use the number of bullet points requested and make each one easy to mark.
