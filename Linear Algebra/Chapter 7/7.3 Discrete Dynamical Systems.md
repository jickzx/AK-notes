## 7.3 Discrete Dynamical Systems

### Definition

A [[7.3 Discrete Dynamical Systems|discrete dynamical system]] updates a state vector step by step using a matrix:

$$
\mathbf{x}_{k+1}=A\mathbf{x}_k.
$$

So after $k$ steps,

$$
\mathbf{x}_k=A^k\mathbf{x}_0.
$$

### Worked Example

Suppose a company tracks the number of users on two plans:

- state $1$: basic plan users
- state $2$: premium plan users

Assume that each month:

- each basic-plan user leads to $1.1$ basic users next month and $0.2$ premium users next month
- each premium user leads to $0.1$ basic users next month and $0.9$ premium users next month

Then the update matrix is

$$
\text{first column = from } {\color{orange} \text{basic users}},
\qquad
\text{second column = from } {\color{cyan} \text{premium users}}
$$

$$
A=
\begin{pmatrix}
{\color{orange} 1.1} & {\color{cyan} 0.1}\\
{\color{orange} 0.2} & {\color{cyan} 0.9}
\end{pmatrix},
\qquad
\mathbf{x}_0=
\begin{pmatrix}
{\color{orange} 100}\\
{\color{cyan} 50}
\end{pmatrix}.
$$

Then

$$
\mathbf{x}_1=A\mathbf{x}_0=
\begin{pmatrix}
{\color{orange} 115}\\
{\color{cyan} 65}
\end{pmatrix},
\qquad
\mathbf{x}_2=A^2\mathbf{x}_0=
\begin{pmatrix}
{\color{orange} \;133}\\
{\color{cyan} \;81.5}
\end{pmatrix}.
$$

So after one month, the model predicts:

- $115$ basic users
- $65$ premium users

After two months, it predicts about:

- $133$ basic users
- $81.5$ premium users

This shows how the state vector changes over time through repeated multiplication by the same matrix.

### Why Eigenvalues Matter

If $A$ is [[Diagonalisation|diagonalisable]], then powers of $A$ are easier to compute:

$$
A^k=PD^kP^{-1}.
$$

That is why [[Eigenvalue|eigenvalues]] and [[Eigenvector|eigenvectors]] are central to long-term behaviour.

### Typical Questions

- Does the system grow, decay, or stabilize?
- Does it approach a steady state?
- Which directions dominate after many steps?
